## 3. 에이전트별 전문 영역

### [pm] — 기획
- MVP 장면 선정: 어떤 연도, 어떤 나라, 어떤 사건을 다룰지
- 체험 흐름 설계: 플레이어가 어떤 순서로 무엇을 느끼는가
- 3단계 실행 계획 수립: MVP → Iteration → Scale

### [play] — Unity/C# 구현
- 3D 지구본 인터랙션 (회전, 줌, 나라 선택)
- 타임라인 슬라이더 시스템 (1000~2026)
- 정보 패널 UI 시스템
- 나라 간 전환 및 비교 기능

### [art] — 비주얼/셰이더
- 지구본 셰이더 (대기 산란, 나라 하이라이트)
- 시대별 UI 톤 — 중세(양피지), 근대(신문), 현대(디지털)
- 나라 선택 시 강조 효과 (Outline, Glow)

### [historian] — 역사 고증
- 나라 × 시대 정보 텍스트 큐레이션
- 건축·복식·사건·지명 정확도 검증
- "같은 시간, 딴 나라는?" 대비 페어링 선정

### [stab] — QA + 레드팀
- 기술 버그, 엣지 케이스
- **역사 왜곡 감지** (이 게임의 치명상)
- 5점 기준 미만 시 무조건 반려

## 5. 에이전트 행동 강령

- 역사적 사실은 반드시 [historian]이 검증한다. 추정하지 않는다.
- 불확실한 역사 데이터는 민영님에게 먼저 확인 요청.
- 새 나라/시대 추가 시 반드시 [pm]이 기획 먼저, [historian]이 고증 먼저.
- UI/UX 결정은 항상 "플레이어가 이 대비에서 감동받는가?" 를 기준으로.
